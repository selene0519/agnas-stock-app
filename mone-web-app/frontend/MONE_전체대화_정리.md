# MONE 앱 개발 전체 대화 정리

> 퀀트 트레이더 관점에서 주식 투자 앱(MONE) 설계 및 구현에 대한 전체 논의 기록

---

## 1. 투자 전략 기준 설계

### 투자 성향 3단계

**보수형 (Conservative)**
- 목표: 손실 가능성을 줄이고 안정적인 종목만 보기
- 손절폭 -2.5% ~ -5%, 목표수익 +5% ~ +10%, 최소 손익비 1.5
- 가중치: riskScore 35%, entryScore 20%, rrScore 15%, momentumScore 15%, qualityScore 10%, newsRisk 감점 5%

**균형형 (Balanced)**
- 목표: 수익 가능성과 리스크를 균형 있게 보기
- 손절폭 -4% ~ -8%, 목표수익 +8% ~ +18%, 최소 손익비 1.8
- 가중치: upsideScore 25%, riskScore 25%, rrScore 20%, momentumScore 15%, entryScore 10%, newsRisk 감점 5%

**공격형 (Aggressive)**
- 목표: 변동성을 감수하고 상승 여력이 큰 종목 보기
- 손절폭 -6% ~ -12%, 목표수익 +15% ~ +30%, 최소 손익비 2.0
- 가중치: upsideScore 35%, momentumScore 25%, rrScore 15%, entryScore 10%, riskScore 10%, newsRisk 감점 5%

### 투자 기간 3단계

**단기 (Short)** — 1~3거래일
- 진입가 거리 ±0~5%, 손절 -2.5~-5%, 목표 +4~+8%
- 중요 지표: 거래대금, 단기 모멘텀, 진입가 터치 가능성

**스윙 (Swing)** — 5~15거래일
- 진입가 거리 ±0~10%, 손절 -5~-8%, 목표 +8~+18%
- 중요 지표: 20일선/60일선, 눌림목 후 회복, 거래량 유지

**중기 (Mid)** — 20~60거래일
- 진입가 거리 ±0~15%, 손절 -7~-12%, 목표 +15~+30%
- 중요 지표: 중기 추세, 재무 안정성, 수급 흐름

### 3×3 조합 의미

| | 단기 | 스윙 | 중기 |
|---|---|---|---|
| 보수 | 가장 안전하고 짧게 | 리스크 낮고 며칠~몇 주 | 안정성 높은 중기 후보 |
| 균형 | 기본 기준의 단기 후보 | **앱의 기본 추천** | 기본 기준의 중기 후보 |
| 공격 | 단기 변동성 큰 급등 후보 | 상승 여력 큰 스윙 후보 | 변동성 감수 큰 목표 |

## 2. 퀀트 트레이더 관점에서 보완 사항

### 리스크 지표 고도화
단순 "변동성 낮음"이 아닌 수치 기준 필요:
- ATR(14일): 절대 변동폭
- Beta vs KOSPI: 시장 대비 민감도
- MDD 20일: 최근 하방 리스크
- 볼린저밴드 폭 (%B): 현재 변동성 상태

**보수형 기준**: ATR 낮고 Beta < 0.8인 종목만
**공격형 기준**: ATR 높고 모멘텀 강한 종목 허용

### 진입가 정량화
"눌림목"을 이동평균 이격도로 정량화:
```
현재가 / 20일선 < 1.03 → 눌림목 진입 가능
현재가 / 20일선 > 1.10 → 추격 진입 경고
```

**기간별 진입 기준선**:
- 단기: 전일 종가 또는 5일 이동평균
- 스윙: 20일 이동평균 또는 최근 5일 저점
- 중기: 60일 이동평균 또는 52주 전저점 반등 구간

### EV(기댓값) 기반 손익비
단순 RR비가 아닌 기댓값 계산:
```
EV = (승률 × 목표수익) - (패율 × 손절폭)
예시: 승률 45%, 목표 +8%, 손절 -4%
EV = 0.45 × 8 - 0.55 × 4 = +1.4% → 진입 가능
```

### 공격형 자동 차단 조건
- 5일 연속 상승 + 거래대금 급감 → 세력 이탈 신호
- RSI > 80 + 볼린저 상단 돌파 → 단기 과열
- 공시 후 갭업 +15% 이상 → 추격 금지

## 3. 눌림목 매수 기준

### 추세 살아있음 확인
```
현재가 > 60일 이동평균   → 중기 상승추세
60일선 > 120일선         → 추세 방향 확인
최근 20일 고점 대비 -5~-15% → 눌림 구간
```

### 눌림목 깊이 기준
- 얕은 눌림 (보수형): 고점 대비 -5% ~ -8%
- 일반 눌림 (균형형): 고점 대비 -8% ~ -15%
- 깊은 눌림 (공격형): 고점 대비 -15% ~ -20%
- -20% 초과 → 추세 붕괴 의심

### 진입 타이밍 확인
```python
recent_high = max(close[-20:])
pullback_pct = (current_price - recent_high) / recent_high * 100

trend_ok = current_price > ma60 and ma60 > ma120
pullback_ok = -15 <= pullback_pct <= -5
volume_ok = volume[-3:].mean() < volume[-20:].mean()  # 눌림 중 거래량 감소
support_ok = current_price >= ma20 * 0.98             # 20일선 근처

is_pullback = trend_ok and pullback_ok and volume_ok and support_ok
```

## 4. 저평가 성장주 기준

### 성장성 기준
- 매출 성장률 YoY > 10%
- 영업이익 성장률 YoY > 15%
- EPS 성장률 YoY > 15%
- 3년 연속 흑자

### 저평가 기준
- PER: 업종 평균 대비 20% 이상 낮음
- PBR: 1.0 이하 또는 ROE 대비 낮음
- **PEG Ratio (핵심)**: PEG = PER / EPS성장률
  - PEG < 1.0 → 저평가 성장주
  - PEG 1~1.5 → 적정
  - PEG > 2.0 → 고평가

### 가치함정 필터
```
부채비율 < 150%         → 재무 안정
영업현금흐름 > 0        → 실제 돈 버는 회사
이자보상배율 > 3        → 이자 갚을 여력
최근 2분기 실적 유지    → 이익 훼손 없음
```

### 스크리닝 5단계
```
1단계: PEG < 1.0 + EPS성장률 > 15%
2단계: 부채비율 < 150% + 현금흐름 양호
3단계: 현재가 / 20일선 비율 0.90~1.02 (눌림목)
4단계: 눌림 중 거래량 감소
5단계: 기관/외국인 수급 확인
```

## 5. 추가 매수 타이밍 전략

### 52주 신고가 돌파
```
52주 고가 대비 현재가 98% 이상 → 돌파 임박
돌파 당일 거래량 > 20일 평균 200% → 진짜 돌파
기관/외국인 동반 매수 → 신뢰도 상승
```
이유: 신고가 = 위에 매물 없음 = 저항 없이 상승 가능

### PEAD 전략 (어닝 서프라이즈 후 조정)
```
EPS 컨센서스 대비 실제 +10% 이상 초과 → 서프라이즈
발표 당일 갭업 후 3~5일 조정 → 눌림목
조정폭 갭의 30~50% 이내 → 추세 유효
```

### 이격도 수렴
```
이격도 = 현재가 / 이동평균선
5일 이격도: 0.97~1.00
20일 이격도: 0.95~1.00  → 최적 진입 구간
60일 이격도: 0.92~1.00
세 이격도 동시 수렴 → 강한 매수 신호
```

### 기관 누적 순매수
```
기관 5일 누적 순매수 전환 → 초기 유입
기관 10일 누적 순매수 지속 → 본격 매집
거래대금 조용 + 기관 순매수 = 아직 안 알려진 것 → 선점 가능
```

### 볼린저밴드 스퀴즈
```
밴드폭 = (상단 - 하단) / 중간선
밴드폭 20일 최저치 → 스퀴즈 상태
상단 돌파 + 거래량 급증 → 상승 방향 확정
```

### 공매도 잔고 감소 (숏커버링)
```
공매도 잔고비율 > 5% → 공매도 많은 종목
공매도 잔고 전주 대비 감소 → 숏커버링 시작
주가 상승 + 공매도 감소 동반 → 강력한 상승 신호
```

### 섹터 로테이션
```
금리 하락 → 성장주, 테크, 바이오
경기 회복 초반 → 소재, 산업재, 반도체
경기 확장 중반 → 소비재, 금융
경기 후반/과열 → 에너지, 원자재
경기 둔화/침체 → 헬스케어, 유틸리티, 배당주
```

## 6. 앱 핵심 기능 설계

### 1. 종목 스크리닝 엔진
- 기본 필터: 시가총액, 거래대금 최소 기준, 관리종목 자동 제외
- 기술적 필터: 이동평균 배열, 이격도, RSI, 볼린저
- 재무 필터: PER/PBR/PEG, 성장률, 부채비율, ROE
- 필터 조합 저장 + 매일 자동 실행 + 결과 푸시 알림

### 2. 매수 타이밍 감지
- 눌림목, 52주 신고가, 볼린저 스퀴즈, 기관 순매수 전환
- 신호 강도 점수화 (★~★★★★★)

### 3. 매도 타이밍 시스템
- 목표가/손절가 알림 (손절 미루기 방지)
- 트레일링 스탑: 고점 대비 -5% 기준 자동 상향
- 시간 기반 청산: 단기 3일, 스윙 15일, 중기 60일 재검토 알림

### 4. 종목 상세 분석
- 기술적: 캔들차트, 이동평균, RSI, MACD, 볼린저
- 수급: 기관/외국인/개인 일별 순매수, 공매도 잔고
- 재무: PER/PBR/PEG, 영업이익, 부채비율, 현금흐름

### 5. 포트폴리오 관리
- 종목별 비중, 섹터 집중도 경고
- 상관관계 높은 종목 경고
- 전략별 수익률 비교, 승률/손익비

### 6. 알림 시스템
- 매수 신호 발생, 목표가/손절가 도달, 기관 대량 매수
- 알림에 이유 포함 필수 (신뢰도 핵심)
- 알림 피로 방지: 중요도 설정, 시간대 설정

### 7. 마켓 레짐 감지
- KOSPI 20일선 위/아래, VIX 수준
- 강세장: 공격형 활성화 / 약세장: 보수형만 / 횡보장: 스윙 중심

### 8. 백테스트
- 전략별 승률, MDD, 샤프 비율, KOSPI 대비 초과수익
- "이 전략이 과거에 실제로 작동했나" → 앱 신뢰도의 핵심

## 7. 결과 보정 및 자기개선 루프

### 결과 추적
```
매수 시점: 종목, 전략 태그, 성향, 기간, 진입/손절/목표가, 신호 종류, 시장 상태
매도 시점: 청산가, 청산 이유, 보유 기간, 최종 수익률
결과 분류: 완전 성공 / 부분 성공 / 손절 / 시간청산 / 기회손실
```

### 전략별 성과 분석
- 9개 조합별 승률/손익비/MDD/샤프 비율 추적
- 신호 종류별 정확도 (눌림목, 신고가, 스퀴즈, 기관순매수)

### 자동 가중치 보정
```python
def adjust_weight(signal_name, recent_win_rate, base_weight):
    if recent_win_rate > 0.60:
        return min(base_weight * 1.3, 0.40)
    elif recent_win_rate < 0.40:
        return max(base_weight * 0.7, 0.05)
    else:
        return base_weight
```

### 피드백 루프 주기
- 실시간: 매도 즉시 기록
- 일간: 신호 정확도 업데이트
- 주간: 가중치 소폭 보정
- 월간: 전략별 비교, 대폭 보정 가능
- 분기: 전체 전략 재검토

### 안전장치
- 최소 30건 이상 누적 후 보정 시작
- 한 번에 ±30% 이내 보정 제한
- 시장 국면 분리 학습 (급락장 데이터로 과도 강화 방지)

## 8. 더 발전시킬 부분

### 앙상블 모델
5개 모델(기술적/수급/재무/모멘텀/뉴스) 투표:
- 5개 동의 → 최강 신호
- 3개 이상 → 일반 신호
- 2개 이하 → 보류

### 머신러닝 레이어 (500건 이상 후)
- 분류 모델: 5일 후 수익/손실 분류
- 설명 가능성 필수 (왜 이 결론인지 근거 표시)

### 시나리오 시뮬레이션
```
강세 시나리오 (30%): 목표 +12%, 8일
기본 시나리오 (50%): 목표 +7%, 12일
약세 시나리오 (20%): 손절 -4%, 5일
EV = 0.30×12 + 0.50×7 - 0.20×4 = +6.3% → 진입
```

### 동적 손절/목표가
- 트레일링 스탑: 고점 대비 -5% 기준 자동 상향
- 1차 목표 달성 시 손절을 진입가로 올려 본전 보호

### 크로스 마켓 연결
- 미국 야간 선물 → 다음날 갭 예측
- AMAT 급등 → 국내 반도체 섹터 선행
- 달러 강세 → 외국인 매도 압력

### 리스크 패리티
```
종목별 비중 = 목표 변동성 / 해당 종목 변동성 (정규화)
→ ATR 낮은 종목 15%, ATR 높은 종목 5%
→ 모든 종목 실제 리스크 노출 동일화
```

### 전략 자동 진화 (유전 알고리즘)
- 파라미터를 "유전자"로 표현
- 여러 변형 버전 백테스트 경쟁
- 성과 좋은 버전이 살아남아 최적 파라미터 자동 발견

## 9. MONE 앱 완전 정의

> MONE은 국장·미장 주식 데이터를 자동 수집하고, 전체 종목을 스캔해, 투자 성향·투자 기간·전략 유형별로 매수 후보를 제안하며, 보유 리스크와 가상운용 성과까지 검증하는 투자 의사결정 보조 앱

**핵심 질문에 답하는 구조:**
- 오늘 어떤 시장을 봐야 하지?
- 지금 매수 가능한 종목이 있나?
- 눌림목 매수 후보는 뭐가 있나?
- 보유종목 중 손절 위험이 있는 종목은?
- 추천이 실제로 맞았나?

## 11. V2 Scoring 패치 내용

### 문제점 (패치 전)
```
보수형 손절: entry × 0.985 = -1.5% (너무 좁음)
공격형 손절: entry × 0.97  = -3%   (근거 없는 고정값)
scoring: 단순 가산 방식 (전략 차별화 없음)
현재가 없으면: PRICE_PENDING으로 scoring 불능
```

### 해결 (패치 후)

**quant_scanner.py — 전략별 가중치 scoring**
```
7개 세부 점수 계산 (0~100):
upsideScore, riskScore, momentumScore, entryScore,
rrScore, qualityScore, newsRiskPenalty

전략별 가중합으로 finalScore 계산
EV(기댓값) 계산 추가
기간별 price band 검증 추가
```

**mone_v65_api_stabilizer.py — 가격 기준 정량화**
```
conservative: stop=0.970(-3%), target=1.060(+6%)
balanced:     stop=0.945(-5.5%), target=1.130(+13%)
aggressive:   stop=0.910(-9%), target=1.200(+20%)

기간별 factor 적용:
short × conservative: 손절 -3.3%, 목표 +4.8%
mid × aggressive:     손절 -11.4%, 목표 +28.6%
```

**quant_scanner.py — OHLCV fallback**
```
현재가 없으면 → OHLCV 최신 close 자동 사용
진입가 괴리 15% 초과 → 현재가 기준 재산출
```

**HomePage.tsx — 세부 점수 시각화**
```
EV(기댓값) 표시 (+x.x%)
전략별 핵심 점수 바 차트
종합점수 숫자 표시
가격대 이탈 경고 뱃지
```

## 13. 주요 주의사항

**lru_cache**: 코드 수정 후 반드시 백엔드 재시작. 재시작 없으면 변경사항 미반영.

**추천 파일 갱신**: `mone_v36_final_recommendations_*.csv`는 GitHub Actions 실행 시만 갱신. 자주 실행할수록 추천 품질 향상.

**진입가 구식화**: 추천 파일의 진입가는 Actions 실행 시점 기준. 현재가와 15% 이상 괴리 시 v2 패치가 자동 재산출.

**holdings_kr.csv**: 보유종목 원장. 앱 조작 시 직접 수정됨. 백업 권장.

**GitHub Actions 무료 한도**: 월 2,000분. 1회 약 5~10분. 월 200~400회 실행 가능.
